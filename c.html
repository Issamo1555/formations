Introduction au Langage PHP

📜 Historique de PHP

PHP (Hypertext Preprocessor) a été créé en 1994 par Rasmus Lerdorf. Évolution majeure :

  • 1995 : PHP/FI 2.0 - Première version publique
  • 1998 : PHP 3.0 - Réécriture complète
  • 2000 : PHP 4.0 - Performance améliorée
  • 2004 : PHP 5.0 - POO complète
  • 2015 : PHP 7.0 - Performance x2
  • 2020 : PHP 8.0 - JIT, Attributes, Union Types

🚀 Points Forts de PHP

Scripts d'Accès aux Données

🔗 Connexion PDO

<?php // Paramètres de connexion $host = 'localhost'; $dbname = 'ma_base'; $username = 'root'; $password = ''; try { $pdo = new PDO( "mysql:host=$host;dbname=$dbname;charset=utf8", $username, $password, [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION] ); echo "Connexion réussie !"; } catch (PDOException $e) { die("Erreur : " . $e->getMessage()); }

📝 Opérations CRUD

// CREATE - Insertion $sql = "INSERT INTO users (nom, email) VALUES (:nom, :email)"; $stmt = $pdo->prepare($sql); $stmt->execute(['nom' => 'John', 'email' => 'john@example.com']); // READ - Lecture $stmt = $pdo->query("SELECT * FROM users"); $users = $stmt->fetchAll(PDO::FETCH_ASSOC); // UPDATE - Mise à jour $sql = "UPDATE users SET email = :email WHERE id = :id"; $stmt = $pdo->prepare($sql); $stmt->execute(['email' => 'new@example.com', 'id' => 1]); // DELETE - Suppression $sql = "DELETE FROM users WHERE id = :id"; $stmt = $pdo->prepare($sql); $stmt->execute(['id' => 1]);

🛡️ Requêtes Préparées

Protection contre les injections SQL :

// ❌ DANGEREUX - Injection SQL possible $pdo->query("SELECT * FROM users WHERE id = $id"); // ✅ SÉCURISÉ - Requête préparée $stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id"); $stmt->execute(['id' => $id]); $user = $stmt->fetch();

Sécuriser les Données

// Créer un cookie sécurisé setcookie( "user_id", // Nom "12345", // Valeur time() + 3600, // Expiration (1h) "/", // Path "", // Domain true, // Secure (HTTPS uniquement) true // HttpOnly (pas d'accès JS) ); // Lire un cookie $userId = $_COOKIE['user_id'] ?? null;

🔐 Sessions

// Démarrer une session session_start(); // Stocker des données $_SESSION['user_id'] = 123; $_SESSION['username'] = 'JohnDoe'; // Vérifier si connecté if (isset($_SESSION['user_id'])) { echo "Bienvenue " . $_SESSION['username']; } // Détruire la session session_destroy();

Web Services & API REST

🌐 Comparaison SOAP vs REST

Critère SOAP REST
Protocole XML strict HTTP + JSON/XML
Performance Plus lourd Léger et rapide
Cache Non Oui

🛠️ Méthodes HTTP REST

// API RESTful en PHP // GET /api/users → Liste tous les utilisateurs // GET /api/users/1 → Récupère l'utilisateur 1 // POST /api/users → Crée un utilisateur // PUT /api/users/1 → Met à jour l'utilisateur 1 // DELETE /api/users/1 → Supprime l'utilisateur 1 // Exemple de contrôleur API class UserApiController { public function create() { $data = json_decode(file_get_contents('php://input'), true); // Traitement... header('Content-Type: application/json'); echo json_encode(['status' => 'success']); } }

📋 Contraintes REST

  • Client-Serveur : Séparation des responsabilités
  • Sans État : Chaque requête contient toute l'information
  • Cacheable : Les réponses peuvent être mises en cache
  • Interface Uniforme : Ressources identifiées par URI
  • Système en Couches : Architecture hiérarchique
